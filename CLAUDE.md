# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Monorepo shape

GovLink is two independent halves under one repo:

```
govlink/
├── backend/    Python 3.12 · FastAPI · SQLAlchemy 2.0 · uv
└── frontend/   React 19 · TypeScript · Vite 8 · Tailwind 3
```

They share nothing at runtime — the frontend talks to the backend over HTTP. Pick the half that matches the task and stay in it; cross-cutting work is **split into two PRs** (one per folder) so reviewers can move independently. There is no root-level package manager, build script, or test runner — every command runs from inside `backend/` or `frontend/`.

## Frontend essentials

The frontend has no dedicated CLAUDE.md, so the non-obvious bits live here.

| Task | Command (run from `frontend/`) |
|---|---|
| Dev server (HMR, proxies `/api`→`:8000`) | `npm run dev` |
| Production build | `npm run build` (runs `tsc -b` then Vite) |
| Type check only | `npm run typecheck` |
| Lint | `npm run lint` |
| Preview built bundle | `npm run preview` |

Stack: React 19, React Router 7, TanStack Query 5, axios, Tailwind 3.4, lucide-react. Path alias `@/*` → `src/*` (configured in both `vite.config.ts` and `tsconfig.app.json`).

Conventions worth knowing:

1. **Decimals are strings.** Backend serialises `Decimal` as JSON strings (`DecimalStr`). Keep them as strings end-to-end; format via `formatDecimalStr` in `src/lib/format.ts`. **Never** `Number(...)` or `parseFloat(...)` them — you'll lose precision on financial data.
2. **All API calls go through `src/lib/api.ts`.** It centralises the axios client, the `{data, meta}` envelope decoding, and the typed `ApiError` class (`isNotFound`, `isRateLimited`). Use `apiGet` / `apiGetEnvelope` / `apiGetRaw` — don't import axios directly from a page.
3. **All queries go through `src/lib/queries.ts`.** New endpoints get a hook there with a key in `queryKeys`, so cache invalidation has one place to live. Pages call the hooks; they don't build URLs.
4. **Vite dev-server proxy.** `/api/*` → `http://localhost:8000` by default; override via `VITE_API_PROXY_TARGET` in `.env`. In production, `VITE_API_URL` controls the axios baseURL.
5. **Theme/tokens.** Visual system is CSS variables in `src/styles/tokens.css` (light + dark) wired through `tailwind.config.ts`. Reference HTML mocks live in `frontend/docs/html/`.
6. **Vercel SPA routing.** `vercel.json` rewrites every path to `/index.html` so client-side routes survive refresh — don't remove that rewrite when adding API routes.

PR checklist before declaring a frontend change done: `npm run typecheck`, `npm run lint`, `npm run build`, and a manual browser pass on the affected pages.

## Tooling that runs automatically (don't fight it)

Hooks defined in `.claude/settings.json` enforce a few invariants on every Edit/Write:

- **`backend/alembic/versions/*.py` edits are blocked.** Migrations are autogenerated via `uv run alembic revision --autogenerate` and ruff is excluded from this directory. If autogen produces something wrong, fix the source models or the autogen run — don't hand-edit the migration file.
- **`.env`, `.env.local`, `.env.*.local` edits are blocked.** Edit them outside Claude; update `.env.example` for documented changes.
- **Post-edit `ruff format` + `ruff check --fix`** runs automatically on any `backend/**/*.py` you touch. No need to invoke it manually after editing.

CI: `.github/workflows/gitleaks.yml` scans every PR and push to `main` for committed secrets. The action is pinned to a commit SHA — bump the SHA and the comment together when updating.

MCP: `.mcp.json` enables the `context7` server for up-to-date library docs. Use it for framework / SDK questions instead of guessing from training data.

## Don't do this

- Don't introduce a top-level `package.json`, root `Makefile`, or any tool that conflates the two halves. They are intentionally independent.
- Don't bypass the backend hooks (no hand-edited migrations, no `.env` rewrites).
- Don't `Number(decimalStr)` in the frontend. Format with `formatDecimalStr`.
- Don't add a frontend dependency on a specific backend dataset slug — pages discover datasets via `/datasets`. The architecture's whole point is that a new dataset shows up without frontend changes.

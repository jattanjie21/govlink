Design system · v1.0

A system for serious public data.

Authoritative, calm, editorial. The design system that powers GovLink — its public site, its admin tools, and every dataset, report and API surface in between.

This guide is the canonical reference for color, type, spacing, motion and voice. Use it as the source of truth for both Figma and code; the values here ship verbatim into our Tailwind config and CSS custom properties.

§ 01

Color

A warm-paper canvas, near-black ink, and a single deep-teal accent. Status colors are muted on purpose — Bootstrap-bright destructives don't belong on a national statistical platform. Every token has a light and a dark value; any new color must extend in OKLch from one of these.

§ 02

Data-viz palettes

Two palettes, both tuned to the canvas. The sequential ramp anchors on --accent for choropleths, heatmaps, and ranked bar charts. The categorical set is for series and legend keys — six colors that are distinguishable in print, in dark mode, and to the most common forms of color blindness.

Order is intentional. Teal is the platform accent — use it for the primary series. Ochre and aubergine sit far enough on the wheel to read against teal in line charts. Rust is the destructive series and is never the primary line. Graphite is for "other" / "all remaining" categories and never the only series.

§ 03

Typography

Fraunces for display — a contemporary serif with editorial weight. Geist for body and UI — modern, neutral, space-efficient. JetBrains Mono for numerics and code. The scale is eight steps from caption to display; do not invent intermediate sizes. Tabular numerals are required wherever data appears.

Layout

Desktop uses a 12-column grid at 1440 with 32-px gutters and 32-px outer padding. Tablet collapses to 8 columns at 768. Mobile is a single column at 375 with 20-px outer padding. The content max-width for long-form prose is 720 px; for dashboard surfaces it's 1200; for editorial-marketing pages it's 1280. Don't fight these.

§ 05

Motion

Restrained. Pages fade in. Cards reveal a thin underline on hover. Nothing bounces. Three durations and two easings cover the entire system.

Duration · fast
--motion-fast: 120ms

Instant feedback. Hover, focus, segment toggle. Anything that needs to feel native.

Duration · med (default)
--motion-med: 180ms

Color and border transitions. Tab content swap. The default if you're not sure.

Duration · slow
--motion-slow: 240ms

Page-load fade-in, dialog open. Use sparingly — slow doesn't mean important.

Easing
--ease-out: cubic-bezier(.2,0,0,1)

Default. Quick start, settled finish — how UI elements settle into place. Linear is reserved for progress and skeleton shimmer only.

The hero stagger on the home and admin pages is 40 ms between elements, capped at the third item. Loaders and skeleton shimmers run at linear. Honour prefers-reduced-motion — reduce all durations to 0 ms and disable the shimmer when set.

§ 07

Do's & don'ts

If something on this list is up for debate, the brief has shifted and the system needs a v1.1 conversation — not a one-off override.

Do

  • Use --accent sparingly. Primary CTAs, link hover, active tab, sequential anchor.
  • Pair Fraunces (display) with Geist (body). Never set body in Fraunces.
  • Tabular numerals everywhere data appears — tables, KPIs, axes, dates, currency.
  • Borders + whitespace before shadows. The system has one shadow token; that's enough.
  • Use rule lines on tables. No zebra striping.
  • Crop to DD MMM YYYY dates and GMD for currency throughout.
  • Show real Gambian context — Banjul, Brikama, GBoS, named ministries — over generic placeholders.

Don't

  • Purple gradient heroes, glassmorphism, or hero video.
  • Rounded "AI-look" cards with a left coloured border accent.
  • Generic emoji feature icons next to every heading.
  • Inter or Roboto as a display face. (Body is fine.)
  • Flag-as-UI. The 24×8 px footer stripe is the only flag reference allowed.
  • Stock imagery of African children, hands holding soil, or other development clichés.
  • Invented metrics — "10× faster", "99.9% uptime" — without a citation.
  • Bouncing micro-interactions, parallax, or animation for its own sake.
§ 08

Accessibility

WCAG 2.2 AA is the floor. Every text/background pair in the token system passes; every status pair passes; the data-viz ramps are tested for legibility on the canvas. The full audit lives in the engineering handoff.

Other floors

  • Hit targets are 44 × 44 px on touch surfaces, 32 × 32 px on dense admin tables.
  • Focus rings are 2 px solid --accent with a 2 px offset. Never remove them; restyle them.
  • Charts must work without color alone — pair the categorical palette with line style or direct labelling. The 5-step ramp must include a legend with numeric breaks.
  • Every form control has a visible label. Placeholder text is not a label.
  • All numbers in tables use tabular-nums and are right-aligned.
